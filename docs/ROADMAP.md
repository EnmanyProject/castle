## 개발 진행 방식

### 각 기능별 워크플로우
```
1. Sam: 기획서 작성 및 요구사항 정의
2. Jenny: 구현 및 기술 문서 작성
3. Will: QA 검토 및 버그 수정
4. 최종 머지 및 버전 태깅
```

### 브랜치 전략
```
master          <- 안정 버전
  └── develop   <- 개발 버전
       ├── feature/quiz-editor
       ├── feature/settings-menu
       └── bugfix/spawn-issue
```

### 버전 규칙
- **Major (x.0.0)**: 대규모 기능 추가
- **Minor (3.x.0)**: 새 기능 추가
- **Patch (3.2.x)**: 버그 수정
